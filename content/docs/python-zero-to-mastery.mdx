## Course vision

This course is designed for two goals at the same time:

* help **you** learn Python from the beginning
* help you **teach others** in a calm, clear, structured way

The main idea is simple:

> do not teach Python as isolated syntax

Teach it as:

* thinking
* problem solving
* small wins
* repetition
* projects

That is what makes a course feel attractive, useful, and memorable.

---

## Who this course is for

This roadmap is ideal for:

* complete beginners
* students who feel afraid of coding
* self-learners who need structure
* teachers who want a clean path from zero to practical Python

You do not need prior programming experience for this course.

---

## What “mastery” means here

Mastery does **not** mean knowing every Python feature.

For this course, mastery means being able to:

* read Python code comfortably
* solve beginner and intermediate problems
* build useful small projects
* explain core concepts clearly
* teach others with confidence

That is real progress.

---

## Teaching philosophy

To make the course beginner-friendly and attractive:

* start with tiny examples
* explain ideas before syntax
* use real-life analogies
* repeat patterns often
* include practice after every concept
* move from simple exercises to mini-projects
* teach students how to think, not what to memorize

This creates a course that feels calm instead of overwhelming.

---

## Recommended course flow

This is the full path:

1. Python mindset and setup
2. Variables and data types
3. Input, output, and operators
4. Conditions and decision making
5. Loops and repetition
6. Functions and reusable logic
7. Lists, strings, tuples, sets, and dictionaries

## Phase 1: Python mindset and setup

### Goal

Help students understand what programming is before they worry about details.

### Teach

* what programming is
* what Python is
* why Python is beginner-friendly
* how code runs
* how to use the terminal or editor
* how to run a simple Python file

### First example

```python
print("Hello, world!")
```

### Explain clearly

This first lesson is not about difficulty.
It is about confidence.

Students need to feel:

> I can write code and make the computer respond

That feeling matters.

### Practice

1. Print your name.
2. Print your age.
3. Print your favorite food.
4. Print three lines of text.

### Teaching tip

Do not rush this part.
The first class should reduce fear.

---

## Phase 2: Variables and data types

### Goal

Teach students how programs store information.

### Core topics

* variables
* strings
* integers
* floats
* booleans
* naming variables

### Example

```python
name = "Sara"
age = 20
height = 1.68
is_student = True

print(name)
print(age)
print(height)
print(is_student)
```

### Key message

A variable is like a labeled box that stores data.

### Practice

1. Create variables for name, city, and favorite color.
2. Store a price and print it.
3. Store whether today is sunny as a boolean.
4. Create five variables and print them all.

### Teaching tip

Use simple personal examples first.
Students connect faster when the data feels familiar.

---

## Phase 3: Input, output, and operators

### Goal

Show students how programs receive information and work with it.

### Core topics

* `input()`
* `print()`
* arithmetic operators
* comparison operators
* converting types with `int()` and `float()`

### Example

```python
name = input("What is your name? ")
age = int(input("What is your age? "))

print("Hello,", name)
print("Next year you will be", age + 1)
```

### Practice

1. Ask for two numbers and print their sum.
2. Ask for a user's name and greet them.
3. Ask for price and quantity, then print the total.
4. Ask for birth year and estimate age.

### Teaching tip

This is the moment students start feeling that code is interactive.
That makes motivation go up quickly.

---

## Phase 4: Conditions and decision making

### Goal

Teach the program to make choices.

### Core topics

* `if`
* `elif`
* `else`
* logical operators: `and`, `or`, `not`
* nested conditions

### Example

```python
age = int(input("Enter your age: "))

if age >= 18:
    print("You are an adult.")
else:
    print("You are under 18.")
```

### Strong beginner examples

* even or odd
* positive, negative, or zero
* pass or fail
* simple login check

### Practice

1. Check whether a number is even or odd.
2. Check whether a score is pass or fail.
3. Check whether a person can vote.
4. Ask for username and print a message if it matches a saved value.

### Teaching tip

Students should say the logic out loud before coding it.

Example:

> If the score is 50 or more, print pass. Otherwise, print fail.

That habit makes coding much easier.

---

## Phase 5: Loops and repetition

### Goal

Teach students how to repeat actions efficiently.

### Core topics

* `for` loops
* `while` loops
* `range()`
* counters
* loop control with `break` and `continue`

### Example

```python
for number in range(1, 6):
    print(number)
```

### Strong beginner examples

* print numbers from 1 to 10
* print even numbers
* count down from 5 to 1
* repeat until a correct password is entered

### Practice

1. Print numbers from 1 to 20.
2. Print all even numbers from 1 to 20.
3. Find the sum of numbers from 1 to 10.
4. Use a `while` loop to keep asking until the user types `"yes"`.

### Teaching tip

Many students struggle here because they cannot visualize repetition.
Manual tracing on the board helps a lot.

---

## Phase 6: Functions and reusable logic

### Goal

Teach students how to organize code into reusable pieces.

### Core topics

* defining functions
* parameters
* return values
* local variables
* why functions matter

### Example

```python
def greet(name):
    return f"Hello, {name}!"

message = greet("Nika")
print(message)
```

### Key message

Functions help us:

* avoid repetition
* organize ideas
* create cleaner code
* explain programs in smaller parts

### Practice

1. Write a function that adds two numbers.
2. Write a function that checks whether a number is positive.
3. Write a function that returns the square of a number.
4. Write a function that greets a user by name.

### Teaching tip

Introduce functions as:

> a machine that receives input, does work, and gives back output

That mental model works very well for beginners.

## Phase 7: Python data structures

### Goal

Teach students how to store and organize collections of data.

### Core topics

* lists
* strings
* tuples
* sets
* dictionaries

### 7.1 Lists

```python
fruits = ["apple", "banana", "orange"]
print(fruits[0])
```

Teach:

* indexing
* appending
* looping through lists
* updating items

### 7.2 Strings

```python
text = "python"
print(text.upper())
print(len(text))
```

Teach:

* string indexing
* common methods
* formatting

### 7.3 Tuples

```python
point = (10, 20)
print(point[0])
```

Teach:

* fixed collections
* when immutability is useful

### 7.4 Sets

```python
numbers = {1, 2, 2, 3, 4}
print(numbers)
```

Teach:

* uniqueness
* membership testing

### 7.5 Dictionaries

```python
student = {
    "name": "Lina",
    "age": 19,
    "grade": 18,
}

print(student["name"])
```

Teach:

* key-value structure
* access and update
* why dictionaries are powerful

### Practice

1. Create a list of five cities and print each city.
2. Count how many characters are in a string.
3. Store student info in a dictionary and print it.
4. Remove duplicates from a list using a set.

### Teaching tip

Do not teach all structures as equal complexity on day one.
Start with lists and strings first, then move to dictionaries and sets.

## Phase 8: Algorithms and problem solving

### Goal

Teach students how to think step by step and solve structured problems.

### Core topics

* input and output thinking
* tracing
* counting
* summing
* finding maximum and minimum
* searching
* beginner patterns

### Example

```python
numbers = [4, 8, 2, 10, 6]
largest = numbers[0]

for number in numbers:
    if number > largest:
        largest = number

print(largest)
```

### Recommended companion lesson

Read and teach [Algorithms From Zero](./algorithms-from-zero.mdx) alongside this phase.

### Practice

1. Count how many numbers are greater than 10.
2. Find the sum of even numbers in a list.
3. Find the smallest number in a list.
4. Check whether a target value exists in a list.

### Teaching tip

This phase is where students stop feeling like they are only learning syntax.
They begin solving real problems.

## Phase 9: Files and error handling

### Goal

Teach students how programs work with files and deal with mistakes safely.

### Core topics

* reading files
* writing files
* `try` and `except`
* common runtime errors

### Example

```python
try:
    with open("notes.txt", "r") as file:
        content = file.read()
        print(content)
except FileNotFoundError:
    print("File not found.")
```

### Practice

1. Write text to a file.
2. Read text from a file.
3. Handle division by zero with `try` and `except`.
4. Handle a missing file error.

### Teaching tip

Students often think errors mean failure.
Teach them that errors are normal feedback from the program.

---

## Phase 10: Modules and code organization

### Goal

Teach students how to split code into multiple files and reuse tools.

### Core topics

* importing modules
* creating your own modules
* using Python standard library tools

### Example

```python
import math

print(math.sqrt(25))
```

### Practice

1. Import `math` and use `sqrt()`.
2. Create a file with helper functions and import it.
3. Build a simple calculator using functions in another file.

### Teaching tip

This is a great point to show students that real programs are organized, not just written in one file.

## Phase 11: Object-oriented programming

### Goal

Introduce classes only after students already understand functions, variables, and data structures.

### Core topics

* classes
* objects
* attributes
* methods
* `__init__`

### Example

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        print(f"My name is {self.name} and I am {self.age} years old.")

student1 = Student("Ali", 21)
student1.introduce()
```

### Practice

1. Create a `Car` class.
2. Create a `Book` class.
3. Create a `BankAccount` class with deposit and withdraw methods.

### Teaching tip

Keep OOP practical.
Avoid making it feel like abstract theory too early.

## A clean teaching structure for each class

Use this class template again and again:

### 1. Warm-up

Review one old concept with 2 or 3 short questions.

### 2. New concept

Explain one idea clearly with one small example.

### 3. Guided example

Solve one problem together step by step.

### 4. Student practice

Give 3 to 5 exercises from easy to medium.

### 5. Reflection

Ask:

* What did we learn?
* What part was confusing?
* Can you explain this idea in your own words?

### 6. Homework or mini-challenge

Give one small task that mixes the new concept with an older one.

This format works extremely well for beginners.

* add two numbers
* check even or odd

### Stage 2 examples

* calculate shopping total

---

## Suggested practice ladder

Each topic should include three levels of exercises:

### Level 1: Direct practice

Use the exact new concept in a simple way.

Example:

> Write a function that returns the double of a number.

### Level 2: Mixed practice

Combine the new concept with older concepts.

Example:

> Write a function that counts how many even numbers are in a list.

### Level 3: Small challenge

Use the concept in a realistic mini-problem.

Example:

> Build a score checker that prints pass, fail, and average.

This creates a healthy learning curve.

## How to make the course feel attractive

Beginner courses become boring when they feel like a dictionary.
To make yours more engaging:

* use friendly lesson titles
* keep sections short and clean
* use example first, theory second when possible
* add “Try this” exercises often
* include mini-projects regularly
* connect ideas to real life
* celebrate practical progress, not just theory

Students return to courses that feel simple, useful, and alive.
