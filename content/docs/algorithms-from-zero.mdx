## What is an algorithm?

An **algorithm** is a **clear sequence of steps** used to solve a problem.

That is all.

You already use algorithms in daily life:

* making tea
* sending a message
* finding a name in a contact list
* checking whether a door is locked

If a task can be broken into steps, it can be described as an algorithm.

> Algorithm = a step-by-step method for solving a problem.

---

## Why should we learn algorithms?

Many beginners think programming is mainly about syntax.
It is not.

Syntax is only the way we write code.
Algorithms are the way we **think**.

When you learn algorithms, you learn how to:

* solve problems in an organized way
* think logically
* avoid random trial and error
* explain your solution clearly
* write better code in any language

---

## A very simple real-life example

Problem:

> You want to find the largest number in a list.

Example list:

```txt
[3, 8, 2, 11, 5]
```

A simple algorithm could be:

1. Start by assuming the first number is the largest.
2. Look at the next number.
3. If the next number is larger, replace your current largest number.
4. Continue until the list ends.
5. The final saved number is the largest.

This is an algorithm before it becomes code.

---

## Algorithms are not the same as code

This is important.

An algorithm is the **idea**.
Code is the **implementation** of that idea.

For example, the following is an algorithm written in plain language:

1. Ask the user for a number.
2. If the number is even, print `"Even"`.
3. Otherwise, print `"Odd"`.

The same idea can be written in Python:

```python
number = int(input("Enter a number: "))

if number % 2 == 0:
    print("Even")
else:
    print("Odd")
```

The algorithm comes first.

## The 4 habits of algorithmic thinking

When solving a problem, train yourself to ask these questions:

### 1. What is the exact problem?

Be precise.

Bad problem statement:

> Work with numbers

Better problem statement:

> Find the sum of all even numbers in a list

The clearer the problem, the easier the algorithm.

### 2. What are the inputs?

What information do we receive?

Examples:

* one number
* two strings
* a list of prices
* a sentence

### 3. What is the expected output?

What should the final answer look like?

Examples:

* `True` or `False`
* a number
* a sorted list
* a message

### 4. What steps transform the input into the output?

This is the algorithm itself.

## A beginner problem solved slowly

Problem:

> Count how many numbers in a list are greater than 10.

Example input:

```txt
[4, 15, 9, 22, 11, 3]
```

Expected output:

```txt
3
```

### Step-by-step thinking

1. Start a counter at `0`.
2. Look at each number one by one.
3. If the number is greater than `10`, increase the counter by `1`.
4. After checking all numbers, print the counter.

### Python version

```python
numbers = [4, 15, 9, 22, 11, 3]
count = 0

for number in numbers:
    if number > 10:
        count += 1

print(count)
```

### Why this example matters

This small problem teaches several important ideas:

* using a variable to store progress
* checking each item in a list
* making decisions with `if`
* repeating work with a loop

Many algorithms are built from these same pieces.

## A teaching pattern that works well

If you want to teach algorithms to others, this order is very effective:

1. State the problem in one simple sentence.
2. Show a small example input and output.
3. Solve it in normal human language.
4. Write the steps as numbered instructions.
5. Translate those steps into Python.
6. Run one example by hand.
7. Give students a similar exercise.

This approach keeps students focused on thinking before coding.

## Example 1: Check whether a number is positive, negative, or zero

Problem:

> Given a number, decide whether it is positive, negative, or zero.

### Algorithm

1. Read the number.
2. If the number is greater than `0`, print `"Positive"`.
3. Else if the number is less than `0`, print `"Negative"`.
4. Otherwise, print `"Zero"`.

### Python

```python
number = int(input("Enter a number: "))

if number > 0:
    print("Positive")
elif number < 0:
    print("Negative")
else:
    print("Zero")
```

### Teaching note

This is a good first algorithm because it introduces:

* comparison
* decision making
* mutually exclusive cases

## Example 2: Find the sum of a list

Problem:

> Add all numbers in a list.

Example:

```txt
[2, 4, 6, 8]
```

Output:

```txt
20
```

### Algorithm

1. Set `total` to `0`.
2. Go through each number in the list.
3. Add the current number to `total`.
4. After the loop ends, print `total`.

### Python

```python
numbers = [2, 4, 6, 8]
total = 0

for number in numbers:
    total += number

print(total)
```

### Manual trace

```txt
Start total = 0
Read 2  -> total = 2
Read 4  -> total = 6
Read 6  -> total = 12
Read 8  -> total = 20
```

Tracing like this is one of the best ways to teach beginners.

## Example 3: Find the maximum number in a list

Problem:

> Find the largest number in a list.

Example:

```txt
[7, 2, 19, 4, 12]
```

Output:

```txt
19
```

### Algorithm

1. Assume the first number is the largest.
2. Compare it with the next number.
3. If the next number is larger, update the largest.
4. Repeat until the list ends.
5. Print the largest number.

### Python

```python
numbers = [7, 2, 19, 4, 12]
largest = numbers[0]

for number in numbers:
    if number > largest:
        largest = number

print(largest)
```

### Important idea

The variable `largest` stores the **best answer found so far**.

This pattern appears in many algorithms:

* smallest value
* highest score
* longest word
* latest date

## Example 4: Search for a value

Problem:

> Check whether the number `8` exists in a list.

Example:

```txt
[3, 5, 8, 10]
```

Output:

```txt
Found
```

### Algorithm

1. Look at each number in the list.
2. If one number equals `8`, print `"Found"` and stop.
3. If the loop finishes without finding `8`, print `"Not found"`.

### Python

```python
numbers = [3, 5, 8, 10]
target = 8
found = False

for number in numbers:
    if number == target:
        found = True
        break

if found:
    print("Found")
else:
    print("Not found")
```

### What students learn here

* searching
* boolean variables
* stopping early with `break`

## A simple class explanation of algorithm categories

At the beginning, students do not need advanced theory.
But they should know that many beginner algorithms fall into a few common groups:

### 1. Decision algorithms

Used when the program must choose.

Examples:

* even or odd
* pass or fail
* positive or negative

### 2. Counting algorithms

Used when we need a total number of something.

Examples:

* count even numbers
* count words longer than 5 letters
* count students with grades above 80

### 3. Accumulation algorithms

Used when values are combined into one result.

Examples:

* sum of numbers
* total price
* average score

### 4. Search algorithms

Used when we need to find something.

Examples:

* find a name
* check whether an item exists
* find the largest number

This categorization makes teaching much easier because students start seeing patterns.

## Common beginner mistakes

Students often struggle not because the problem is hard, but because one detail is unclear.

Here are the most common mistakes:

### 1. Starting to code before understanding the problem

Fix:

> First explain the problem in plain language.

### 2. Forgetting the initial value

Examples:

* `count = 0`
* `total = 0`
* `found = False`

Initial values matter a lot.

### 3. Mixing up one item with the whole list

For example:

* `numbers` is the full list
* `number` is one item from the list

### 4. Not tracing the algorithm by hand

Manual tracing helps students discover logic errors early.

### 5. Memorizing code without understanding the pattern

Students should learn the idea:

* initialize
* loop
* check or update
* print the result

## A classroom-friendly problem-solving template

This template is useful for both self-study and teaching.

### Step 1: Problem

Write the problem in one sentence.

Example:

> Find how many numbers are even in a list.

### Step 2: Input

Write the input clearly.

Example:

```txt
[1, 2, 3, 4, 5, 6]
```

### Step 3: Output

Write the expected result.

Example:

```txt
3
```

### Step 4: Human solution

Explain how a person would solve it.

Example:

> Check each number. If it is divisible by 2, increase the counter.

### Step 5: Algorithm

Write exact steps.

1. Set `count` to `0`.
2. Loop through each number.
3. If the number is even, add `1` to `count`.
4. Print `count`.

### Step 6: Python code

```python
numbers = [1, 2, 3, 4, 5, 6]
count = 0

for number in numbers:
    if number % 2 == 0:
        count += 1

print(count)
```

### Step 7: Trace

```txt
1 -> odd  -> count = 0
2 -> even -> count = 1
3 -> odd  -> count = 1
4 -> even -> count = 2
5 -> odd  -> count = 2
6 -> even -> count = 3
```

This structure is excellent for teaching.

## Practice section for students

Use these in class, in homework, or for self-study.

### Level 1: Very basic

1. Write an algorithm to check whether a number is even or odd.
2. Write an algorithm to print numbers from `1` to `10`.
3. Write an algorithm to find the sum of three numbers.
4. Write an algorithm to count how many numbers in a list are positive.

### Level 2: Core beginner practice

1. Find the smallest number in a list.
2. Count how many words in a list have more than 4 letters.
3. Check whether a given name exists in a list of names.
4. Find the sum of only the even numbers in a list.

### Level 3: Good teaching exercises

1. Find the average of a list of numbers.
2. Count how many students passed if pass mark is `50`.
3. Find the longest word in a list.
4. Count how many times a target value appears in a list.

## Mini classroom activity

This is a strong teaching activity for a live class.

### Activity: Human algorithm

Ask one student to act as the "computer".
Give the student a list of numbers on the board:

```txt
[5, 12, 7, 18, 3]
```

Then ask the class to give exact instructions for:

> Find how many numbers are greater than 10.

Rules:

* the "computer" only follows exact instructions
* vague instructions are not allowed
* each step must be clear

This makes students feel why algorithms must be precise.

## How to explain algorithms beautifully when teaching

If you want the lesson to feel clean and attractive, keep this teaching style:

* use one small problem at a time
* use short lists instead of huge inputs
* explain before coding
* trace values line by line
* repeat the same pattern in different problems
* move from easy to slightly harder

Students learn faster when the lesson feels calm, clear, and predictable.

---

## Final mental model

When students see a problem, they should start thinking like this:

1. What is the input?
2. What is the output?
3. What should I store in a variable?
4. Do I need `if`?
5. Do I need a loop?
6. What changes during each step?
7. What should I print at the end?

If they can answer those questions, they can build many beginner algorithms.

---

## Final definition

An algorithm is:

> a clear, ordered set of steps that transforms input into output to solve a problem.

That idea is one of the foundations of programming.

Learn it slowly, practice it often, and teach it with small examples.

That is the strongest path for beginners.
