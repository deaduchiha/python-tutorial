# Four key things to learn in any programming language

When someone starts programming, it is easy to feel buried under syntax, tools, and new vocabulary.
The better way to learn is to focus on a few foundational ideas first.

In almost any programming language, there are **four key things** you need to understand:

* **language terms**
* **data types**
* **actions or operations**
* **best practices**

If you understand these four areas clearly, learning the rest becomes much easier.

---

## 1. Language terms

Every programming language has its own vocabulary.
Before writing bigger programs, you need to become comfortable with the basic terms people use when talking about code.

Some of the most important beginner terms are:

* statements
* variables
* expressions
* functions
* classes
* objects
* object instantiation

Let’s go through the most important ones carefully.

### Statements

A **statement** is an instruction that tells Python to do something.

Examples:

```python
print("Hello")
x = 10
name = "Nika"
```

Each line above is a statement.
It tells the program to perform an action.

You can think of a statement as:

> one meaningful instruction in a program

### Variables

A **variable** is a name that refers to a value.

Example:

```python
age = 21
```

Here:

* `age` is the variable name
* `21` is the value

Variables let us store information so we can use it later.

Examples:

```python
name = "Sara"
score = 95
is_logged_in = True
```

Variables are one of the first big ideas in programming because programs need a way to remember data while they run.

### Expressions

An **expression** is a piece of code that produces a value.

Examples:

```python
2 + 3
"Hi" + " there"
10 > 5
```

Each of these gives back a result:

* `2 + 3` becomes `5`
* `"Hi" + " there"` becomes `"Hi there"`
* `10 > 5` becomes `True`

Expressions are important because many statements contain expressions inside them.

Example:

```python
total = 4 + 6
```

Here, `4 + 6` is an expression, and the whole line is a statement.

### Functions

A **function** is a reusable block of code that performs a task.

Example:

```python
def greet(name):
    print(f"Hello, {name}")
```

Functions help us:

* avoid repetition
* organize code
* give names to actions

### Classes and objects

A **class** is a blueprint for creating objects.
An **object** is a real instance created from that blueprint.

Example:

```python
class Student:
    def __init__(self, name):
        self.name = name
```

This class describes what a `Student` object should look like.

### Object instantiation

**Object instantiation** means creating an object from a class.

Example:

```python
student1 = Student("Ali")
```

Here:

* `Student` is the class
* `student1` is an object created from that class

For beginners, the main thing to remember is:

> a class is the design, and an object is the actual thing created from that design

## 2. Data types

Data is at the center of programming.
Programs receive data, store data, change data, compare data, and return data.

To work correctly, Python needs to know what kind of value it is dealing with.
That is where **data types** come in.

A **data type** defines what kind of value something is and how it can be used.

For example:

* numbers can be added
* text can be joined
* lists can store multiple items
* booleans can represent true or false states

In Python, every value has a type.

### Built-in (fundamental) data types in Python

Some of the most common built-in Python types are:

* `int`
* `float`
* `bool`
* `str`
* `list`
* `tuple`
* `set`
* `dict`

These are enough to build a huge number of beginner and intermediate programs.

Let’s understand them one by one.

### `int`

`int` means **integer**.
It is used for whole numbers.

Examples:

```python
age = 20
year = 2026
temperature = -5
```

Use `int` when you need:

* counting
* indexes
* ages
* whole-number calculations

### `float`

`float` means a number with a decimal point.

Examples:

```python
price = 19.99
height = 1.75
score = 87.5
```

Use `float` for:

* measurements
* prices
* averages
* values that are not whole numbers

### `bool`

`bool` means **boolean**.
It has only two possible values:

* `True`
* `False`

Examples:

```python
is_active = True
is_admin = False
```

Booleans are used in conditions and decisions.

Example:

```python
if is_active:
    print("Account is active")
```

### `str`

`str` means **string**, which is text.

Examples:

```python
name = "Nika"
message = "Welcome to Python"
```

Strings are used for:

* names
* messages
* user input
* text processing

### `list`

A `list` stores multiple values in order.

Example:

```python
numbers = [1, 2, 3, 4]
fruits = ["apple", "banana", "orange"]
```

Lists are useful when:

* order matters
* you want to add or remove items
* you need to loop through many values

### `tuple`

A `tuple` is similar to a list, but it is typically used for data that should not change.

Example:

```python
point = (10, 20)
rgb = (255, 0, 0)
```

Tuples are useful when the data is fixed.

### `set`

A `set` stores unique values.

Example:

```python
tags = {"python", "beginner", "coding"}
```

Sets are helpful when:

* duplicates should be removed
* you only care whether a value exists

Example:

```python
numbers = {1, 2, 2, 3}
print(numbers)
```

Output:

```txt
{1, 2, 3}
```

### `dict`

A `dict` stores data as **key-value pairs**.

Example:

```python
student = {
    "name": "Sara",
    "age": 21,
    "grade": 19,
}
```

Dictionaries are powerful because they let us store structured information.

This makes them useful for:

* user profiles
* settings
* product details
* records with labels

---

## Why data types matter

Data types matter because they affect:

* what operations are allowed
* how values are stored
* how Python interprets your code

For example:

```python
print(2 + 3)
print("2" + "3")
```

These look similar, but they behave differently:

* `2 + 3` gives `5`
* `"2" + "3"` gives `"23"`

Why?
Because the data types are different.

This is one of the first big lessons in Python:

> the type of a value changes what Python does with it

---

## Custom types with classes

Python does not only give you built-in types.
You can also create your own **custom types** using **classes**.

Example:

```python
class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author

book1 = Book("Clean Code", "Robert C. Martin")
print(book1.title)
```

Here, `Book` is a custom type created by the programmer.

This is important because real-world programs often need to model things like:

* students
* products
* bank accounts
* orders
* cars

Built-in types are not always enough on their own.

Classes let you design types that match the problem you are solving.

## 3. Actions and operations

After learning what values are, the next step is learning what we can **do** with them.

Programming is not just about storing data.
It is about acting on data.

Common actions in Python include:

* storing data
* retrieving data
* transforming data
* comparing data
* combining data
* repeating actions
* making decisions

Let’s look at these more clearly.

### Storing data

We store data in variables.

Example:

```python
username = "nika"
```

This lets us keep a value and reuse it later.

### Retrieving data

We retrieve data when we access stored values.

Example:

```python
name = "Lina"
print(name)
```

Or from a collection:

```python
colors = ["red", "blue", "green"]
print(colors[1])
```

### Transforming data

We transform data when we change it into another form.

Examples:

```python
text = "python"
print(text.upper())

age = "20"
print(int(age) + 5)
```

In the first example, lowercase text becomes uppercase.
In the second example, text becomes a number.

### Comparing data

We often compare values in order to make decisions.

Examples:

```python
print(10 > 5)
print("apple" == "apple")
print(7 != 3)
```

These operations return `True` or `False`.

### Combining data

We can combine values in different ways.

Examples:

```python
full_name = "Ali" + " " + "Reza"
total = 5 + 10
```

### Repeating actions

Programs often need to repeat work.
That is where loops come in.

Example:

```python
for number in [1, 2, 3]:
    print(number)
```

### Making decisions

Programs also need to choose between different paths.

Example:

```python
age = 18

if age >= 18:
    print("Adult")
else:
    print("Minor")
```

When beginners understand these actions well, code becomes much less mysterious.

## 4. Best practices

Best practices are habits and conventions that make code:

* easier to read
* easier to test
* easier to debug
* easier to maintain
* easier to teach

A beginner does not need to know every advanced software engineering rule.
But some habits are important from the start.

### Use clear variable names

Good:

```python
student_name = "Mina"
total_price = 50
```

Less clear:

```python
x = "Mina"
y = 50
```

Names should help people understand the purpose of the value.

### Keep code simple

Do not try to impress people with complicated code.
Simple code is usually better code.

Good beginner rule:

> if a shorter and clearer way exists, prefer it

### Be consistent

Choose a style and follow it.

Examples:

* use meaningful naming
* use the same indentation style
* keep similar code structured similarly

Consistency makes code feel professional.

### Write code for humans, not only for computers

Python can run messy code if the syntax is valid.
But other people may struggle to read it.

Code should be understandable to:

* your future self
* your students
* your teammates

### Break big problems into smaller parts

Instead of writing everything in one long block, use:

* variables
* functions
* clean steps

This helps both learning and teaching.

### Test your ideas with small examples

Beginners improve faster when they test code in tiny pieces.

Example:

```python
number = 4
print(number % 2 == 0)
```

Before solving a large problem, test the small logic first.

### Accept errors as part of learning

Errors are not proof that you are bad at programming.
They are normal feedback.

Strong beginners learn to ask:

* What does the error say?
* Which line caused it?
* What type of value am I using?
* What did I expect to happen?

That mindset is much more useful than panic.

## Modules in Python

Modules are not a built-in data type like `int` or `str`, but they are a very important part of Python.

A **module** is a Python file that groups related code together.

Modules help us organize programs and reuse functionality.

For example, Python already includes many useful standard modules.

Example:

```python
import math

print(math.sqrt(25))
```

Here:

* `math` is a module
* `sqrt()` is a function inside that module

Modules are useful because they:

* keep code organized
* reduce repetition
* let you reuse existing tools
* make large programs manageable

You can also create your own modules.

Example:

`helpers.py`

```python
def greet(name):
    return f"Hello, {name}"
```

`main.py`

```python
from helpers import greet

print(greet("Sara"))
```

This is one of the first steps toward writing cleaner multi-file programs.

## Understanding `None`

`None` is one of the most important values for beginners to understand.

`None` represents the **absence of a value**.

It often means:

* nothing is here yet
* no result was found
* no value was provided

Example:

```python
result = None
print(result)
```

Output:

```txt
None
```

### Why `None` matters

Programs often need a special value to represent “empty” or “not available yet.”

Example:

```python
selected_user = None
```

This means:

> no user is selected at the moment

### `None` is not the same as `0` or `""`

This is very important.

* `0` is a number
* `""` is an empty string
* `[]` is an empty list
* `None` means no value

These are not the same idea.

Example:

```python
print(0 == None)
print("" == None)
```

Both comparisons are `False`.

### Checking for `None`

The best practice is:

```python
if value is None:
    print("No value yet")
```

Using `is None` is preferred over `== None`.

## A beginner mental model for Python

If you want a clean mental model, think of Python like this:

* **variables** store values
* **data types** describe those values
* **operations** act on those values
* **statements** tell the program what to do
* **functions** organize reusable logic
* **classes** define custom objects
* **modules** organize code across files
* **best practices** make everything clearer and safer

This model is simple, but it is powerful.

## How to study these topics well

If you are learning Python from zero, study in this order:

1. variables and statements
2. built-in data types
3. operators and basic actions
4. conditions and loops
5. functions
6. lists and dictionaries
7. modules
8. classes and objects
9. best practices through small projects

Do not try to master everything in one day.
The goal is not speed.
The goal is clarity.

## Final summary

To learn Python well, you should not only memorize syntax.
You should understand how the language is built and how programmers think inside it.

The strongest foundation includes:

* language terms like statements, variables, and objects
* data types like numbers, strings, lists, and dictionaries
* actions like storing, retrieving, transforming, and comparing data
* best practices that make code readable and reliable
* important ideas like modules and `None`

If these ideas become clear, the rest of Python becomes much easier to learn, use, and teach.
